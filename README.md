# Hi, I'm Ailun Pei 👋

**Software Engineer — backend systems by day, rendering and interactive tools by night.**

At **Polainco** I own the order-ingestion and inventory-sync backend for a live-commerce operation: **Java, Spring Boot, MySQL, Redis, AWS**, end-to-end from schema to deploy.

Outside of work I build things that run in a browser — learning tools, shader playgrounds, audio visualizers. Everything below is live, not a screenshot.

## Selected Work

Three projects, all deployed and running.

### 🧠 [NoteFlow](https://github.com/Dollars7/NoteFlow) · [Live](ADD_URL)

A retention-first learning app. Turns Markdown notes into retrieval cards and decides what you should review next.

`React` `TypeScript` `Cloudflare Workers` `D1`

Built the scheduling engine and the edge data layer — [ADD: e.g. sub-100ms card retrieval, N cards scheduled across M decks]. The interesting problem isn't storing notes, it's deciding what's worth resurfacing.

---

### ✨ [WebGL Shader Playground](https://github.com/Dollars7/WebGL-Shader-Playground) · [Live](ADD_URL)

Real-time shader playground with custom GLSL pipelines, material controls, and dynamic lighting. Holds 60fps with live recompilation.

`WebGL` `GLSL` `JavaScript`

Wrote the render loop, the uniform binding layer, and the shader hot-reload path — including the part nobody enjoys, which is surfacing GLSL compile errors in a way that's actually readable.

---

### 🎵 [Music Visualizer Studio](https://github.com/Dollars7/music-visualizer-studio) · [Live](ADD_URL)

Real-time audio visualization: browser-side WebGL rendering wired to an FFmpeg media-processing pipeline.

`WebGL` `GLSL` `JavaScript` `FFmpeg`

FFT analysis feeding shader uniforms every frame, with an offline export path for rendering to video. Getting audio and video to agree on what "now" means was most of the work.
